# Why DECODE Switched to Claude

> Internal · Last updated: April 2026

---

## The Short Answer

We switched from individual Claude Pro accounts to **Claude Team** because we needed shared infrastructure — not just a personal AI tool, but a platform the whole team works on together.

---

## 1. Shared Projects

With Claude Team, we can create **Public Projects** that are visible and accessible to the entire organisation.

This means:
- One project = one shared context, one shared knowledge base
- Everyone works with the same instructions, the same tone guidelines, the same documents
- No more copy-pasting prompts between colleagues
- New team members can start immediately without setup

**Example:** The "DECODE Texts" project has our tone guidelines and writing style pre-loaded. Everyone who opens it gets the same Claude — already briefed on how we write.

---

## 2. Team Features & Admin Control

Claude Team gives us central control over how Claude is used across the organisation.

As admin, Kleo can:
- **Enable or disable features** for the whole org (e.g. web search, feedback button)
- **Manage connectors** — no one can activate a new integration without approval
- **Control data privacy settings** centrally — one setup, applies to everyone
- **Monitor usage** across the team

This means every team member works within the same guardrails — without having to manage their own settings.

---

## 3. From Pro to Team — What Changed

| | Claude Pro (before) | Claude Team (now) |
|---|---|---|
| Projects | Personal only | Shared across the org |
| Settings | Per person | Centrally managed |
| Connectors | Self-managed | Admin-controlled |
| Knowledge base | Personal | Shared per project |
| Usage monitoring | Personal only | Org-wide |

---

## Bottom Line

Claude Team is not just Claude with more seats — it's Claude as a **shared team infrastructure**. That's why we switched.
